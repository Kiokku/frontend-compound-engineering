# 前端功能规划

## 输入
<feature_description>#$ARGUMENTS</feature_description>

## 工作流

### 1. 设计分析
解析设计稿(Figma/Sketch/原型图)：
- 识别页面结构和布局模式
- 标记交互元素和状态变化
- 提取设计 token（颜色、字体、间距）
- 记录响应式断点需求

### 2. 组件规划
识别可复用组件和原子组件：
- 创建组件树结构图
- 标识公共组件（Button, Input, Modal 等）
- 规划业务组件（Header, Footer, Sidebar 等）
- 确定组件 Props 接口

### 3. 技术选型
根据项目框架选择实现方案：
- 确认前端框架（React/Vue/Angular/Svelte）
- 选择状态管理方案
- 确定样式解决方案（CSS Modules/Tailwind/Styled-components）
- 评估第三方组件库需求

### 4. 依赖分析
列出需要安装的 npm 包：
- 核心框架依赖
- UI 组件库
- 工具库（日期处理、表单验证等）
- 开发依赖（类型定义、构建工具）

### 5. 测试策略
单元测试、集成测试、E2E 测试：
- 确定测试框架（Jest/Vitest/Playwright）
- 规划组件单元测试
- 设计集成测试场景
- 定义 E2E 关键路径

### 6. 性能预估
首屏加载、交互响应时间目标：
- 设定 LCP（最大内容绘制）目标 < 2.5s
- 设定 FID（首次输入延迟）目标 < 100ms
- 设定 CLS（累积布局偏移）目标 < 0.1
- 规划代码分割和懒加载策略

## 输出

生成以下交付物：

1. **计划文件**: `plans/<feature-name>.md`
   - 功能概述
   - 技术方案
   - 实施步骤
   - 时间估算

2. **组件清单**
   - 组件名称
   - 组件层级关系
   - Props 接口定义
   - 依赖关系

3. **依赖清单**
   - package.json 需要添加的依赖
   - 版本建议
   - 安装命令
